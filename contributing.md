# Contributing Guide

## Branching Strategy

- **main**: Stable production code.
- **dev**: Integration branch for new features.
- **feature/<name>**: Individual feature branches.

## Pull Request Process

1. Create a feature branch.
2. Commit changes with clear messages.
3. Push to GitHub and open a Pull Request.
4. Request at least one review before merging.

## Code Review Guidelines

- Check for functionality and clarity.
- Ensure adherence to PEP8 style.
- Validate that tests pass.

## Commit Message Conventions

- Use present tense: "Add LD pruning function".
- Prefix with type: `feat:`, `fix:`, `docs:`.
